Best practices for using classes & ids

In CSS, the .class and #id selectors can be used to style specific HTML elements. In this post, we’ll take a quick look at how these selectors are used, and describe the best practices for when to use classes instead of id’s, and vice-versa.

.class selector

The class selector is used in CSS for styling multiple HTML elements. An HTML element can also inherit CSS attributes from multiple classes!

If you want to apply your style to any HTML element, use a period symbol followed by the class name. For example:

CSS:

                    
    .purple-text {
        color: purple;
    }

HTML:

                    
    <h1 class="purple-text”>This is a purple title</h1>

    <p>
        Wow, that is a <span class="purple-text”>purple</span> word!
    </p>

Result:

This is a purple title

Wow, that is a purple word!

Notice how the class style was applied to both the h1 element and the word "purple"? Cool, huh?

element.class selector

You can be a little more specific with your class selector. Say you wanted to apply your style to multiple HTML elements, but wanted some variance depending on the element type. We can specify the element first, before the period symbol and the class name!

CSS:

                    
    .purple-text {
        color: purple;
    }

    h1.purple-text {
        text-transform: uppercase;
    }

    span.purple-text {
        font-family: 'Kaushan Script';
    }

Let's apply that to the same HTML as before!

Result:

This is a purple title

Wow, that is a purple word!

Sweet, both the h1 element and the word "purple" have had their colour applied from the .purple-text class. But even sweeter: additional CSS properties have been applied specifically for their element type!

#id selector

The id selector is used when styling a single HTML element. Each element can only have one id.

As a best practice, the id should not be reused on any other element (make it a class instead!) However, an id can still be used in conjunction with one or more classes.

The id selector is used by a hash symbol, followed with the id name. For example:

CSS:

                    
    .purple-text {
        color: purple;
    }

    h1.purple-text {
        text-transform: uppercase;
    }

    span.purple-text {
        font-family: 'Kaushan Script';
    }

    #gold-text {
        color: white;
        background-color: gold;
    }

HTML:

                    
    <h1 class="purple-text”>This is a purple title</h1>

    <p>
        Wow, that is a 
        <span id="gold-text" class="purple-text”>
            purple
        </span> 
        word!
    </p>

It's just the original HTML from the start! But this time, we've specified the id attribute in addition to the class attribute.

Result:

This is a purple title

Wow, that is a purple word!

Okay, that is clearly not a purple word... But notice how the .purple-text class is still an attribute on the <span> HTML tag? That means the #gold-text id has taken priority and overridden the .purple-text class property.

The last thing worth pointing out: the span.purple-text class is still applied (this changes the font specifically for <span> elements). That's because it doesn't conflict with the #gold-text id - by that I mean they are both altering different CSS properties, so there's nothing to override.



I hope that I've been able to share a bit of insight into the best practices for using the .class and #id selectors. And, when one might be more useful than the other.
